AWS Services for DevOps Engineers

1. Compute Services

  • EC2 (Elastic Compute Cloud): Launch virtual machines (VMs), manage instance types, auto-scaling, load balancing, and spot instances.
  • Lambda: Serverless computing for event-driven architectures without provisioning servers.
  • ECS (Elastic Container Service): Containerized applications using Docker.
  • EKS (Elastic Kubernetes Service): Running Kubernetes clusters in AWS.
  • Elastic Beanstalk: PaaS for deploying applications with minimal management.

2. Storage Services

  • S3 (Simple Storage Service): Object storage for backups, logs, or static content.
  • EBS (Elastic Block Store): Block storage for EC2 instances.
  • EFS (Elastic File System): Managed file storage for sharing across EC2 instances.
  • Glacier: Low-cost storage for archiving and long-term backups.

3. Networking & Content Delivery

  • VPC (Virtual Private Cloud): Configuring private networks, subnets, route tables, and security groups.
  • Route 53: DNS and routing for low-latency traffic routing.
  • Elastic Load Balancer (ELB): Distributing traffic across multiple instances (Classic, Application, Network).
  • CloudFront: Content Delivery Network (CDN) for low-latency global content distribution.

4. Automation & Infrastructure as Code

  • CloudFormation: Define and deploy AWS resources using templates.
  • Terraform: Infrastructure as code tool commonly used with AWS services.
  • OpsWorks: Configuration management with Chef or Puppet.
  • AWS Systems Manager (SSM): Automating EC2 management, commands, patching, and tasks.

5. CI/CD & DevOps Tools

  • CodePipeline: Automates the release process with CI/CD workflows.
  • CodeCommit: Source control, similar to GitHub or Bitbucket.
  • CodeBuild: Builds source code, runs tests, and produces artifacts.
  • CodeDeploy: Automates deployment to EC2, Lambda, and other services.
  • Elastic Container Registry (ECR): Docker image registry for containerized applications.

6. Monitoring & Logging

  • CloudWatch: Monitoring, logging, and setting up alarms for AWS resources.
  • CloudTrail: API call and AWS service activity logging for security and auditing.
  • X-Ray: Debugging and tracing microservices.
  • AWS Config: Tracks resource configurations for compliance auditing.

7. Security & Identity

  • IAM (Identity and Access Management): Managing user access and permissions.
  • KMS (Key Management Service): Encryption keys for data protection.
  • Secrets Manager: Store and manage sensitive data like API keys and passwords.
  • AWS WAF (Web Application Firewall): Protect web apps from common threats.
  • GuardDuty: Threat detection and monitoring for malicious activity.

8. Databases

  • RDS (Relational Database Service): Managed relational databases like MySQL, PostgreSQL.
  • DynamoDB: Managed NoSQL database for high-performance applications.
  • ElastiCache: In-memory data store (Redis, Memcached) for caching.
  • Aurora: Highly available, fully managed relational database.

9. High Availability & Disaster Recovery

  • Route 53 Health Checks: Failover routing to another region during disaster.
  • S3 Cross-Region Replication: Replicate S3 data across regions for high availability.

10. Cost Management

  • AWS Cost Explorer: Monitor and manage AWS spending.
  • AWS Budgets: Set custom spending budgets and track usage.
  • AWS Trusted Advisor: Best practices and cost optimization recommendations.

11. Migration Services

  • AWS DMS (Database Migration Service): Migrate databases to AWS.
  • AWS SMS (Server Migration Service): Migrate on-premises workloads to AWS.

12. Application Services

  • Step Functions: Orchestrate microservices workflows.
  • SNS (Simple Notification Service): Send notifications via SMS, email, etc.
  • SQS (Simple Queue Service): Decouple microservices with message queues.
  • AppSync: Managed GraphQL service for building serverless apps.
